📖 Manual de Especificaciones Técnicas y de Diseño
Proyecto: "Escritorio del Archivista" 
Este sistema está diseñado para que archivistas e historiadores carguen inventarios en formato de hoja de cálculo (Excel/CSV) y documentos PDF maestros digitales de gran volumen (libros antiguos escaneados), con el fin de fragmentar el PDF en archivos individuales por cada registro histórico. Su valor principal reside en la validación en tiempo real de la consistencia física de la foliación correlativa (detectar folios perdidos, saltos, hojas en blanco o errores de transcripción antes de procesar).

V. Algoritmos Clave y Lógica del Frontend (Core Logic)
Para asegurar que la consistencia física funcione, el frontend debe implementar la siguiente lógica de análisis correlativo:
1. El Intérprete de Folios (Parser Heurístico)
Un folio histórico se define generalmente por un número de folio y una cara (r = recto, v = verso). Un rango se expresa como 002r-003v.
La otra IA debe implementar una función que convierta estos strings en números enteros planos para poder calcular secuencias matemáticas correlativas:
code
Python
# Ejemplo de lógica en Python para interpretar la foliación:
import re

def parse_folios(folio_str: str):
    folio_str = folio_str.strip().lower()
    
    # Expresión regular para rangos como '002r-003v'
    rango_match = re.match(r'(\d+)([rv])?-(\d+)([rv])?', folio_str)
    if rango_match:
        desde_num = int(rango_match.group(1))
        desde_cara = rango_match.group(2) if rango_match.group(2) else 'r'
        hasta_num = int(rango_match.group(3))
        hasta_cara = rango_match.group(4) if rango_match.group(4) else 'r'
        return desde_num, desde_cara, hasta_num, hasta_cara
        
    # Expresión regular para folios únicos como '140' o '140r'
    unico_match = re.match(r'(\d+)([rv])?', folio_str)
    if unico_match:
        num = int(unico_match.group(1))
        cara = unico_match.group(2) if unico_match.group(2) else 'r'
        return num, cara, num, cara
        
    return None # Error de formato
2. Algoritmo de Detección de Saltos de Secuencia
La aplicación debe recorrer los registros de manera secuencial y comprobar que el folio inicial del registro actual corresponde exactamente al folio final del registro anterior más la secuencia natural de páginas (recto -> verso -> siguiente página recto):
Si el número de folio no coincide y la regla de exclusión no tiene registrado un salto aprobado para ese rango, el sistema debe cambiar automáticamente el estado del registro a REVISAR, levantar una sugerencia de corrección con el valor esperado calculado y emitir un log tipo WARN en la consola interactiva.
